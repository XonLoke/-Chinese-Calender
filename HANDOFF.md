## Session Context

The Chinese Calendar Engine (chinese_calendar) is a zero-dependency Python library for accurate Chinese calendar calculations, built on Dershowitz & Reingold and Meeus algorithms. **Sprints 1-8 are all completed** (v0.8.0): core RataDie, VSOP87 sun engine, 24 solar terms, new moon calculation, Chinese lunisolar calendar, ganzhi system, caching layer, bilingual locale, and Calendar API.

The engine is functionally complete but has **two categories of remaining work**: (1) installing PyMeeus and running the full test suite to validate astronomical accuracy, and (2) building the web testing interface. Future modules (遁甲历, 飞星历, 年卦, 建除十二神等) are planned but not yet started.

## Technical Context

### Current State
- **Language**: Python 3.10+
- **Dependencies**: Zero runtime (PyMeeus optional for astronomy)
- **Architecture**: 4-layer stack (core → astronomy → calendar → api/locale)
- **Version**: v0.8.0 (Sprint 8 complete)
- **Tests**: 230+ tests, 74 skip without PyMeeus

### Key Decisions
| Decision | Rationale |
|---|---|
| Rata Die as universal bridge | Every calendar converts to/from R.D., never directly |
| PyMeeus for VSOP87 | Faster to ship; VSOP87 self-implement reserved as fallback |
| Winter solstice anchors November | D&R Ch.4 algorithm: find冬至, then locate 11th month |
| Circular dependency via lazy import | `time_systems` ↔ `converters` ↔ `chinese` cycle |

### Architecture
```
chinese_calendar/
├── core/           rata_die.py, time_systems.py, constants.py
├── astronomy/      sun.py, moon.py, solar_terms.py, corrections.py
├── calendar/       converters.py, chinese.py, ganzhi.py
├── data/           cache.py, precompute.py
├── locale/         types.py, zh_CN.py, en_US.py
├── api/            solar_date.py, lunar_date.py, calendar_api.py
├── tests/          (11 test files)
└── CLAUDE.md       Full project guide (load this first)
```
